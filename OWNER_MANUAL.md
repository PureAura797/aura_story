# PureAura: Руководство по управлению контентом (Owner's Manual)

Данное руководство описывает структуру проекта после внедрения **Фазы 3 (Next.js App Router)** и содержит инструкции о том, как изменять контент на сайте без помощи разработчика.

## 1. Структура проекта (Где что лежит)

Все визуальные компоненты и страницы находятся в директории `src/`:

*   **`src/app/`** — Здесь настроена маршрутизация (страницы сайта).
    *   `page.tsx` — Главная страница.
    *   `about/page.tsx` — Страница "О компании".
    *   `uslugi/[имя_услуги]/page.tsx` — Внутренние страницы услуг (например, уборка после смерти, накопительство).
*   **`src/components/sections/`** — Готовые блоки, из которых состоят страницы.
    *   `Services.tsx` — Блок с карточками услуг (Bento grid) на главной.
    *   `Expertise.tsx` — Блок "Почему нельзя доверять клинерам" (тексты об угрозах).
    *   `Pricing.tsx` — Блок с ценами и тарифами.
    *   `Hero.tsx` — Первый экран главной страницы с заголовком и фоном.
    *   `ContactForm.tsx` — Форма обратной связи.
*   **`src/components/layout/`** — Глобальные компоненты (Шапка и Подвал).

## 2. Как изменять тексты на главной странице

### 2.1 Изменение текста в карточках услуг (Главная)
Откройте файл `src/components/sections/Services.tsx`. Прокрутите до HTML-кода (в секции `return`). Вы увидите блоки `<Link href="..." className="bento-card...">`.
Внутри карточки вы можете легко найти русский текст, например:
```tsx
<h3 className="text-2xl font-medium mb-3 group-hover:text-teal-300 transition-colors">Уборка после смерти</h3>
<p className="text-neutral-400 text-[15px] max-w-md leading-relaxed">
  Мы берем на себя тяжелое бремя и юридические риски... 
</p>
```
*Инструкция:* Просто замените текст внутри тегов `<h3>` или `<p>` на ваш. Ничего больше менять не нужно. Сохраните файл, и сайт обновится автоматически.

### 2.2 Изменение цен и тарифов (Блок Pricing)
Откройте файл `src/components/sections/Pricing.tsx`.
В самом начале файла вы найдете массив `pricingPlans`. Он выглядит так:
```tsx
const pricingPlans = [
  {
    name: "Очаговая дезинфекция",
    price: "От 15 000 ₽", // <-- МЕНЯТЬ ЗДЕСЬ
    description: "Локальная обработка места происшествия (до 5 кв.м).",
    features: [
      "Выезд бригады в течение 60 минут",
      "Удаление видимых биозагрязнений",
      // ...
    ]
  },
  // ...
]
```
*Инструкция:* Для изменения цены просто поменяйте строку `"От 15 000 ₽"`. Вы также можете добавлять или удалять строки из списка `features`.

### 2.3 Изменение блока "Почему мы" (Expertise)
Откройте файл `src/components/sections/Expertise.tsx`.
Аналогично ценам, здесь есть массив `expertiseItems`:
```tsx
const expertiseItems = [
  {
    icon: Biohazard,
    title: "Скрытая биологическая угроза", // <-- Заголовок карточки
    description: "Видимая кровь — лишь 10% проблемы...", // <-- Текст
    // ...
  }
]
```

## 3. Как изменять контент на внутренних страницах услуг

Внутренние страницы (например, уборка после смерти) находятся в `src/app/uslugi/uborka-posle-smerti/page.tsx`.

Внутри вы найдете SEO-метатеги (title и description), которые используются для поискового продвижения Яндексом:
```tsx
export const metadata: Metadata = {
  title: "Уборка после смерти и дезинфекция помещений в Москве | PureAura",
  description: "Профессиональная уборка и дезинфекция квартир после смерти...",
};
```
*Инструкция:* Крайне аккуратно меняйте эти поля, так как они напрямую влияют на SEO. Делайте их привлекательными для системного поиска.

Весь остальной текст на странице скомпонован в виде стандартного HTML-текста (в React использующего `className`), вы можете смело переписывать русскую часть текста, не затрагивая сами теги `<h1 className="...">` или `<p>`.

## 4. Развертывание (Деплой)

Когда вы внесете изменения:
1. Если вы разрабатываете локально: `npm run dev` покажет вам изменения на `localhost:3000`.
2. Когда все готово к публикации: скомандуйте `npm run build`. Это оптимизирует проект и соберет готовые статические файлы.
3. Платформа хостинга (например, Vercel или Яндекс.Облако) обычно автоматически подхватит изменения из GitHub после коммита.

## 5. Добавление новой услуги
Если вам потребуется добавить новую услугу, выполните 3 шага:
1. Создайте папку в `src/app/uslugi/новая-услуга/`.
2. Создайте в ней файл `page.tsx` (можно просто скопировать код из другой услуги и заменить тексты).
3. Добавьте карточку со ссылкой `/uslugi/новая-услуга` в `src/components/sections/Services.tsx`.
